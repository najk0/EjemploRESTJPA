The project

Given the complexity of the project we chose to follow the Scrum methodology, which has aided development by allowing us to work through prototypes rather than trying to do everything at once. This methodology also helped us discard ideas that ended up being either too complex or irrelevant.

Technologies and standards

qWiki is a web application that follows a RESTful JAX-RS based architecture for Java. The RESTful architecture is built upon the Jersey framework, and a MEAN stack (MongoDB, Express, Angular2 and Node) is used to provide an actual and dynamic front-end user experience. Finally Wikipedia's API is used as the main source of information, and an automatic summarization library (Classifier4J) summarizes its contents. We also performed version control throughout the project using GitHub.

Work methodology

This project has followed the Scrum agile methodology, which consists in dividing the duration of the project into short development cycles (usually 2 weeks) called sprints. Each sprint provides a final product increment (prototype) that can be delivered to the customer for review.

In the planning of each sprint, the user stories that will be covered are decided. After planning, the team, composed of 4 people, meets every day before starting to work to catch up on the work of colleagues and resolve doubts or problems that have arisen since the last meeting.

Once the sprint is finished, a meeting is held to review the sprint and, if necessary, show the resulting prototype to the customer.

Financing
